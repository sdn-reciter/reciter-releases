# ESpeech сервер — лёгкий качественный русский (файнтюн F5-TTS)

TTS-сервер на [ESpeech-TTS-1](https://huggingface.co/ESpeech/ESpeech-TTS-1_RL-V2)
— русском файнтюне F5-TTS (обучен на 4000+ часов русской речи, Apache 2.0).
API полностью совпадает с остальными серверами (`/tts`, `/accent`, `/voices`,
`/health`) — приложение переключается **сменой адреса/пресета**:

| Сервер | Порт | Голоса | Вес модели |
|---|---|---|---|
| XTTS-v2 | 8002 | клоны по wav | ~1.8B |
| Supertonic-3 | 8003 | пресеты + Voice Builder | 99M |
| VoxCPM2 | 8004 | клоны по wav | 2B |
| **ESpeech (F5 ru)** | **8005** | клоны по wav | **~0.3B** |

## Чем интересен

- **Русский — родной**: файнтюн именно под русский, а не мультиязычная модель
  «в том числе с русским». Заметно меньше акцента и ошибок ударений.
- **Ударения «+» — родной формат**: модель обучена на метках `+` перед
  ударной гласной (тех самых, что ставит RUAccent). Сервер по умолчанию сам
  проставляет ударения перед синтезом (`ESPEECH_SYNTH_STRESS=plus`) — это
  главный вклад в качество.
- **Лёгкая**: ~0.3B (в ~6 раз меньше VoxCPM2) — на 8 ГБ GPU живёт с большим
  запасом, быстрее реального времени.
- **Клонирование по wav** — те же эталоны, что у XTTS/VoxCPM.

## Особенность F5: референсу нужен транскрипт

F5-архитектура требует референс **всегда** (без него синтез невозможен) и
вместе с текстом этого референса. Сервер решает это сам:

1. Референс длиннее `ESPEECH_REF_MAX_SEC` (12 с) обрезается ffmpeg'ом, и
   транскрипт берётся **от обрезанного куска** (кеш в `voices/.refcache`).
   Аудио и текст обязаны соответствовать друг другу: по их соотношению F5
   оценивает темп речи — транскрипт полного файла при обрезанном аудио дал бы
   сверхбыстрое чтение и «хвосты» референса на стыках предложений.
2. Короткий референс (≤ 12 с) идёт как есть; рядом ищется `<голос>.txt` —
   положи его руками для идеальной точности.
3. Если `.txt` нет — **один раз** распознаёт речь faster-whisper'ом (CPU) и
   кеширует.

Совет: лучший эталон — 8–12 с ЧИСТОЙ речи одного диктора без музыки и
рекламных интонаций; длинные «демо-нарезки» дают нестабильный тембр.

## Запуск

```bash
# 0. Один раз: общий базовый образ (CUDA+torch+RUAccent) — см. ../base/README.md
docker build -t reciter-tts-base:cu128 ../base

mkdir -p /mnt/data/reciter-tts/espeech-server && cd /mnt/data/reciter-tts/espeech-server
# скопируйте сюда файлы этой папки
cp .env.example .env

# Эталоны: скопируй готовые из XTTS/VoxCPM (каталог не может быть пуст!)
mkdir -p voices && cp /mnt/data/reciter-tts/xtts-server/voices/*.wav voices/ 2>/dev/null || true

docker compose build && docker compose up -d   # веса качаются один раз при сборке
docker compose logs -f
```

Проверка:

```bash
curl http://localhost:8005/health
curl -X POST http://localhost:8005/tts \
  -F 'text=Привет! Это тест русского файнтюна Эф пять.' -F 'language=ru' \
  -o test.wav && aplay test.wav
```

## Переключение в приложении

Настройки чтения → «Свой TTS-сервер» → адрес `http://<IP-ПК>:8005` (или через
Tailscale `http://<имя-ПК>:8005`) → «Сохранить как…» пресет `ESpeech`.
Голоса из `./voices` появятся и в выпадающем списке пикера голоса.

## Настройки качества/скорости (.env)

```env
ESPEECH_NFE=32          # 16 — почти вдвое быстрее, чуть проще звук
ESPEECH_SYNTH_STRESS=plus  # родные ударения модели — НЕ выключай без причины
ESPEECH_MODEL=ESpeech/ESpeech-TTS-1_podcaster  # вариант с «подкастерской» подачей
```

После правки `.env`: смена модели — `docker compose build && docker compose
up -d`; остальное — `docker compose up -d --force-recreate`.

## Откат

Отдельный контейнер: `docker compose down` — XTTS (8002), Supertonic (8003)
и VoxCPM (8004) не затрагиваются.
